# 灵伴 - 陪伴机器人监控面板

这是一个为"灵伴"陪伴机器人设计的前端监控面板，模拟天天的智能玩偶与用户的交互界面。

## 故事背景

- **天天**: 一个可爱的小女孩，她是灵伴的小主人
- **灵伴**: 天天的陪伴机器人，是一个智能玩偶，能识别天天并与她互动

## 功能特性

- **人脸识别显示**: 实时显示摄像头画面，识别当前访客是"天天"还是"陌生人"
- **灵伴虚拟形象**: 可爱的机器人动画形象，展示灵伴的状态变化
- **智能对话**:
  - 识别到天天 → "你好呀天天，现在想做什么"
  - 识别到陌生人 → "你是谁呀"
- **飞书通知**: 当天天说"输液快完了"，自动显示通知护士的提示
- **状态日志**: 记录所有交互历史
- **对话记录**: 展示天天和灵伴的对话

## 快速开始

### 1. 启动前端服务

```bash
cd frontend/public
python -m http.server 3000
```

访问：http://localhost:3000

### 2. 配置设置

点击右上角 ⚙️ 图标打开设置面板：

- **WebSocket 地址**: 连接 xiaozhi-server (默认: `ws://localhost:8080`)
- **摄像头地址**: ESP32 摄像头地址 (默认: `http://192.168.4.1:81/stream`)

### 3. 点击连接

连接成功后，灵伴会说"你好！我是灵伴~"

## WebSocket 消息协议

### 接收消息

```javascript
// 人脸识别结果
{ "type": "identity", "identity": "tiantian" | "stranger" }

// 语音识别结果 (天天说的话)
{ "type": "stt", "text": "输液快完了" }

// AI 响应 (灵伴说的话)
{ "type": "llm", "text": "你好呀天天，现在想做什么", "emotion": "happy" }

// 语音合成状态
{ "type": "tts", "state": "start" | "stop" }

// 飞书通知
{ "type": "notification", "action": "nurse_called", "message": "通知护士拔针" }
```

### 发送消息

```javascript
// 握手
{ "type": "hello", "version": "3", "client_id": "lingban-web-xxx" }
```

## 特殊场景

### 识别天天
当 OpenClaw 识别到镜头前是天天时：
- 左侧显示"✅ 识别成功: 天天"
- 当前访客卡片显示"天天"和"小主人回来啦~"
- 灵伴说"你好呀天天，现在想做什么"

### 识别陌生人
当识别到不是天天的人时：
- 左侧显示"⚠️ 未知访客"
- 当前访客卡片显示"陌生人"和"我不认识你..."
- 灵伴说"你是谁呀"

### 输液提醒
当天天说"输液快完了"或类似内容时：
- 界面显示飞书通知面板
- 显示"通知护士前来拔针"

## 依赖

- 无需额外依赖，纯原生 HTML/CSS/JavaScript

## 故障排除

### 摄像头无法显示
1. 确认 ESP32 摄像头已正确配置并开启
2. 检查摄像头地址是否正确
3. 浏览器直接访问摄像头地址测试

### WebSocket 连接失败
1. 确认 xiaozhi-server 已启动
2. 检查 WebSocket 地址
3. 查看浏览器控制台错误信息
